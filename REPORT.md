## 1. Описание концепции

### Идея и настроение

Концепция **«Университет у моря»** в визуальном языке актуального sochi.university: тёмный navy/violet hero, magenta CTA, белые «парящие» панели и линейные синие иконки.

Ключевые визуальные приёмы:
- **Тёмный градиентный первый экран** (navy → violet) с фото студентов и blueprint-графикой кампуса.
- **Magenta CTA** («Поступить») + ghost-кнопка («Смотреть видео»).
- **Плавающая белая панель** уровней образования (бакалавриат → ДПО) с линейными иконками.
- **Ряд преимуществ** с синими line-иконками на светлом фоне.
- Типографика **Montserrat**, иерархия как у современных вузовских сайтов (UTMN + фирменный стиль СочиГУ).

### Макеты

Отдельный Figma-файл в рамках этой поставки не приложен: **рабочим макетом служит сама вёрстка**.

| Версия | Как посмотреть |
|--------|----------------|
| Desktop 1440×900 / 1920×1080 | Открыть `index.html`, ширина окна ≥1200px |
| Mobile 375×812 | DevTools → iPhone / ширина 375px |

Рекомендуется снять скриншоты этих двух вьюпортов и приложить к отклику, если работодатель ждёт PDF/Figma отдельно.

## 3. UX-анализ текущей главной sochi.university

### Что мешает

1. **Перегруженный первый экран** — баннер приёма + попап подписки + служебная плашка «тестовый режим» конкурируют за внимание.
2. **Слабый фокус на абитуриенте** — CTA есть, но путь «выбрать программу → подать документы → контакты ПК» не собран в один сценарий.
3. **События размыты** — длинная лента памятных дат смешивается с реальными университетскими мероприятиями.
4. **Иерархия заголовков** — несколько H1 на странице (баннеры) ухудшают доступность и SEO.
5. **Мобильная навигация** — тяжёлое меню и много конкурирующих блоков усложняют быстрый доступ к поступлению.

### Что улучшено в концепции

| Проблема | Решение на новой главной |
|----------|---------------------------|
| Размытый hero | Один заголовок + lead + 2 CTA + 3 цифры |
| Нет сценария поступления | Блок «Как поступить» в 4 шага + контакты ПК |
| Шумные события | 3 релевантных мероприятия вместо длинной ленты дат |
| Слабая семантика | Один H1, секции `header/main/section/article/footer` |
| Сложно найти программы | Сетка популярных направлений с иконками |

### Технические требования

- Адаптив: 320–480 / 768 / 1200+
- Семантика HTML5
- Базовая доступность: skip-link, alt/aria, контраст, логичные заголовки, `prefers-reduced-motion`
- JS: меню, якоря, active state, reveal-on-scroll, кнопка «наверх»

### Ограничения (честно)

- Нет отдельного файла Figma/PSD — концепция описана здесь, визуал = HTML.
- Фото героя — стоковое (Unsplash), в проде лучше заменить на фото кампуса/студентов СочиГУ.
- Новости/события статичны (без CMS), контент взят/адаптирован с официального сайта.
- Официальный логотип СочиГУ заменён на упрощённый SVG-знак в фирменных цветах (для тестового макета).
